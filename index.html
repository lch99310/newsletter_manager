Setup Guide / 設定教學
Click the Fork button at the top-right of the GitHub repo page. This creates your own copy where GitHub Actions will run on your behalf.
Go to resend.com and create a free account. In the dashboard, go to API Keys and create a new key.

Free tier: 100 emails/day, 3,000/month — more than enough for daily newsletter checks.
In your forked repo, go to Settings → Secrets and variables → Actions. Add these two secrets: 
RESEND_API_KEY  = your Resend API key
FROM_EMAIL     = onboarding@resend.dev
Note: Use onboarding@resend.dev for testing. Once you verify a custom domain on Resend, you can use your own email address.
Forked repos have Actions disabled by default. Go to the Actions tab and click "I understand my workflows, go ahead and enable them".
Edit data/sources.json to add websites: 
{
  "sources": [
    {
      "url": "https://www.latepost.com/",
      "name": "LatePost (晚点)"
    },
    {
      "url": "https://example.com/blog",
      "name": "Example Blog"
    }
  ]
}
Edit data/subscribers.json to add your email: 
{
  "subscribers": [
    "your-real-email@gmail.com"
  ]
}
Use the Edit on GitHub buttons in the Configuration section above for quick access.
Go to the Actions tab, click "Check for Updates" workflow on the left, then click "Run workflow" dropdown on the right and hit the green button.

Check the workflow logs to see results. If new articles are found, you'll receive an email.
